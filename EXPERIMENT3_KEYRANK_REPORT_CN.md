# 实验3复现报告：9轮 Key-Rank 统计（攻击质量量化）

## 数据来源

由于当前环境无法联网安装 `tensorflow/keras/numpy` 依赖，先基于仓库自带的官方补充数据复现实验3统计结果：

- `supplementary_data/data_9r_attack/9r_attack_keyrank_32.npy`
- `supplementary_data/data_9r_attack/9r_attack_keyrank_64.npy`
- `supplementary_data/data_9r_attack/9r_attack_keyrank_128.npy`

以上文件由论文作者随仓库提供，对应 9 轮攻击在不同密文对数量（32/64/128）下的 key-rank 样本（各 1000 次）。

## 核心结果

### 32 对密文
- 样本数：1000
- 均值：575.415
- 中位数：38
- P90/P95/P99：1231 / 2768 / 8789
- Top-1（rank<=0）：10.00%
- Top-2（rank<=1）：16.30%
- Top-16（rank<=15）：39.90%

### 64 对密文
- 样本数：1000
- 均值：52.072
- 中位数：1
- P90/P95/P99：36 / 108 / 893
- Top-1：35.80%
- Top-2：50.80%
- Top-16：82.80%

### 128 对密文
- 样本数：1000
- 均值：0.860
- 中位数：0
- P90/P95/P99：2 / 4 / 11
- Top-1：71.50%
- Top-2：87.20%
- Top-16：99.60%

## 现象解读（用于讲解）

1. 随着可用密文对从 32 增加到 64，再到 128，key-rank 显著下降（越小越好）。
2. 128 对时，中位数已经到 0，表示多数实验里真实密钥排名已在第一位。
3. Top-K 成功率随数据量快速上升，说明神经区分器有效压缩了密钥搜索空间。
4. 分布有长尾（例如 32 对时 P99 很高），说明少量困难样本仍可能失败，但总体趋势稳定。
